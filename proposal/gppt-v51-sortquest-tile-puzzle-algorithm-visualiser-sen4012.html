1. Executive Summary

This proposal presents SortQuest: Tile Puzzle Algorithm Visualiser (SEN4012), a Digital Puzzle Game project targeting SEN4012 Analysis of Algorithms students (3rd/4th year, n≈35) in the domain of academic. The project is grounded in Flow Theory (Csikszentmihalyi, 1990), Self-Determination Theory (Deci & Ryan, 1985), MDA Framework (Hunicke et al., 2004), employing Points, Leaderboard, progress, challenge as primary mechanics.

2. Problem Statement

Students understand sorting algorithm names but cannot trace execution on a given array. SortQuest makes sorting visual and tactile: drag tiles into correct positions following real algorithm rules. Wrong individual moves are rejected immediately with an algorithm-specific explanation — there is no path to a correct final state through random dragging.

7. Theoretical Framework

SDT Analysis

Autonomy: Students choose their approach to each sorting puzzle independently — selecting which sorting algorithm to apply, determining step sequence, and deciding when to use hint tokens versus reasoning independently. The open sandbox mode allows students to freely experiment with algorithm variants without scoring pressure, supporting self-directed algorithm exploration beyond the structured puzzle chain.

Competence: Each puzzle completion with a correct sorting algorithm application awards XP proportional to the solution's time complexity optimality, making algorithm knowledge directly rewarding in a quantifiable way. The puzzle difficulty ladder (Bubble Sort fundamentals to Heap Sort) provides an explicit, topic-aligned competence staircase. Incorrect solutions reveal the execution step where the chosen algorithm diverged from correct behaviour.

Relatedness: Post-session puzzle board shared with the class shows which students reached which difficulty levels, creating peer conversation about sorting algorithm trade-offs. Group speed-sort challenges (simultaneous puzzle solving with live class comparison) create shared competitive energy. Pair mode allows two students to collaboratively solve a sorting challenge, requiring negotiated algorithm selection.

MDA Analysis

Mechanics: Puzzle chain (sequential sorting challenges), Algorithm selection panel, Step-by-step execution validation, Hint token system (3 per session), Sandbox mode (unscored experimentation), Leaderboard by difficulty level reached, Speed-sort challenge mode.

Dynamics: The step-by-step execution validation mechanic forces students to think at the algorithm's execution level rather than just knowing the algorithm name, producing deeper cognitive engagement. Hint token scarcity creates authentic cost-benefit reasoning about knowledge confidence. Sandbox mode provides a low-stakes exploration space that complements the scored puzzle chain.

Aesthetics: Challenge (each puzzle requires genuine algorithm application, not guessing), Discovery (sandbox mode enables free algorithm experimentation), Accomplishment (advanced difficulty puzzle completion signals genuine SEN4012 mastery), Submission (puzzle aesthetic creates engaging spatial representation of abstract sorting concepts).

Selected frameworks: Flow Theory (Csikszentmihalyi, 1990) · Self-Determination Theory (Deci & Ryan, 1985) · MDA Framework (Hunicke et al., 2004) · Bloom's Taxonomy (Bloom et al., 1956)

10. Narrative Framework

Theme: The SortQuest realm is in chaos — the Library of Ordered Knowledge has been scrambled by the Entropy Demon. As the last Algorithm Keeper, restore order using the ancient sorting laws.

World: The Sorting Academy is an ancient institution where computational knowledge is tested not by memorisation but by execution. Six halls represent the six sorting algorithm families: the Bubble Hall, the Selection Chamber, the Insertion Corridor, the Merge Sanctum, the Quick Arena, and the Heap Pinnacle. Each hall tests a specific algorithm family with increasing input complexity and edge-case difficulty. The Academy Boss Puzzle chamber sits at the pinnacle — accessible only to students who have demonstrated mastery across all six halls.

Game Overview Summary

Executive Summary: A tile-puzzle visualiser for SEN4012 where students physically sort array tiles using a chosen algorithm — each move costs XP proportional to its operation count.

Purpose: Make abstract sorting algorithms concrete through physical manipulation with real operation-count cost feedback.

What is Taught: Sorting algorithms: bubble, insertion, merge, quick, heap, counting — Levitin Chapters 3-7

How Learning is Achieved: Flow: puzzle complexity scales with n; students experience O(n²) vs O(n log n) cost difference through XP expenditure.

Measurement: Pre/post quiz improvement ≥20% measured 48h after session; L3 transfer assessed via next assignment grade

Players: SEN4012 Year-3/4 CE students, n≈32

Points / XP System: +50 XP for sort completion; bonus XP for fewer-than-optimal moves; leaderboard by total operations

Achievements: Merge Master: complete merge-sort with 0 extra swaps; Speed Sorter: beat theoretical minimum step count

Badges: Sort Expert badge: complete all 6 sorting algorithm puzzles; Optimal Sorter: achieve theoretical minimum moves

    Learning Content Details & Teaching Methodology

    Instructor Role: The instructor should only be assigned the role of questioning the game. No additional roles beyond questioning should be assigned. The instructor's focus is on facilitating reflection, probing understanding, and guiding critical thinking — not participating in game mechanics. If the instructor participates as a player, grading objectivity and the psychological safety of students may be compromised.

    Game Duration Rules: No single game should exceed 30 minutes of active gameplay. Total session including introduction and debrief: 30 minutes. Recommended timing: Introduction 5 min + Active gameplay 20 min + Debrief & reflection 5 min. Instructor may pause at any time for teaching moments. Games exceeding time limit are concluded at the current phase boundary.

    Teaching Subject: SEN4012: Algorithm complexity analysis, Big-O notation, sorting, searching, dynamic programming, greedy, graph algorithms — Levitin Chapters 2-13

    Declarative Knowledge: Pre-challenge briefings define algorithm properties and complexity classes using in-game visual explanations

    Procedural Knowledge: Students apply algorithm selection decisions with immediate consequence feedback (correct = wave cleared; wrong = wave penalty)

    Methodology: Active learning via consequence-driven decision-making: challenge → decision → immediate feedback → reflection → next challenge

    Game Dynamics
    Progression Systems

    Puzzle chain advances through six sorting algorithm families in complexity order: Bubble Sort, Selection Sort, Insertion Sort, Merge Sort, QuickSort, and Heap Sort. Each family contains 3 puzzles of increasing input size and edge-case complexity. Completing a family unlocks the next and awards a family badge. The final boss puzzle requires choosing and applying the optimal algorithm for a complex mixed-requirement scenario.

    Feedback Loops

    Step-validation feedback highlights the exact execution step where a student's algorithm trace diverges from correct behaviour, providing precise actionable feedback. Hint token use is tracked and displayed in the post-session report, providing feedback on knowledge confidence patterns. Sandbox mode provides self-directed feedback through observable algorithm execution without scoring pressure.

    Structured Competition

    Leaderboard by puzzle chain depth and efficiency (hints used per puzzle). Weekly Sort Master badge for the student with the deepest chain and fewest hints used. Speed-sort challenge mode (all students attempt the same puzzle simultaneously with live class timer) creates high-energy competitive events throughout the semester.

    Collaboration Dynamics

    Pair mode pairs students on a shared puzzle: one selects the algorithm, one executes the steps. Role alternation per puzzle ensures both students engage both selection and execution competencies. Disagreements on algorithm selection trigger a collaborative reasoning phase before commitment to the solution path.

    Adaptability

    Puzzle difficulty and input complexity are configurable per session. Hint token allocation is adjustable for assessment vs practice contexts. Sandbox mode can be enabled or disabled per session. Boss puzzle can be swapped per session to cover different algorithm category combinations. Speed-sort challenge timing is adjustable.

    Social Interaction

    Collaboration

    Pair mode assigns algorithm selection to one student and execution trace to the other — creating complementary collaboration that ensures both students engage both cognitive layers of sorting algorithm mastery.

    Competition

    Class speed-sort challenge mode creates simultaneous competitive puzzle solving visible to all students — turning individual algorithm practice into a shared competitive performance event.

    Community Building

    Post-session class puzzle board showing depth reached and hints used creates transparent peer comparison that drives improvement across sessions; sandbox findings are shared in class debrief.
